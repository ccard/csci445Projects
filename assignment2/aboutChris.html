Current Work

My current thesis work involives using computer vision to perform indoor localization on mobile devices. This is difficult because the indoor environment has sparse local image texture features for example:


This requires the use of line segment intersections and this can be done throught the following code:
bool MyLine::lineIntersection(MyLine other,Point2f& i){
Mat l1 = lineEq();
Mat l2 = other.lineEq();
Mat p = l1.cross(l2);
pair<Point2f,Point2f> points1 = getPoints();
pair<Point2f,Point2f> points2 = other.getPoints();
Point2f p1_1 = points1.first;
Point2f p1_2 = points1.second;
Point2f p2_1 = points2.first;
Point2f p2_2 = points2.second;
float p_div = (p.at<float>(2,0) == 0 ? 1.f : p.at<float>(2,0));
Point2f p_3(p.at<float>(0,0)/p_div,p.at<float>(1,0)/p_div);

bool fits = (min(p1_1.x, p1_2.x) <= p_3.x) && (max(p1_1.x, p1_2.x) >= p_3.x);
fits &= (min(p1_1.y, p1_2.y) <= p_3.y) && (max(p1_1.y, p1_2.y) >= p_3.y);
fits &= (min(p2_1.x, p2_2.x) <= p_3.x) && (max(p2_1.x, p2_2.x) >= p_3.x);
fits &= (min(p2_1.y, p2_2.y) <= p_3.y) && (max(p2_1.y , p2_2.y)>= p_3.y);
i = (fits ? p_3 : Point2f(-1.f,-1.f));
return fits;
}

These line segment intersections can be used to calculate the cross-ration of 4 colinear points. This cross ration is invariant to scale and perspective making it ideal to use as matching method in images that may not be from the same perspective. If there are any further questions please email me.
